객체 지향 프로그램 시작 
---------------------

객체 - 클래스 - 객체 

java.lang.Object root 클래스이다. 
계층형 구조 가지고 있다. : 부모 - 자식 
상속 구조를 가지고 있다. : inheritance
자바에서의 상속 구조는 자식이 부모의 자원을 사용할 수 있다 또 변경해서 사용할 수 있다. 
단 부모는 자식의 자원을 사용하려면 자식에게 허락을 받아야 한다. 

자바에서 상속
	단일 상속 : extends 키워드 : 클래스 상속, 인터페이스에서 인터페이스 상속할 때 
	다중 상속 : implements 키워드 : 클래스에서 인터페이스를 상속할 때


클래스 종류
--------------------
일반 클래스 public class 사용자정의 클래스이름 
인터페이스 클래스 public interface 사용자정의 인터페이스 이름
추상 클래스  public abstrct class 사용자정의 클래스이름 
파이날 클래스 public final class 사용자정의 클래스이름 

클래스 발전

일반 클래스 -> 추상클래스 -> 인터페이스 

자바에서 클래스를 만드는 이유 : 자원을 사용하려고 : 변수 + 함수 

클래스 사용 : static, new, 상속(단일 상속)
추상클래스 : 일반함수, 추상함수
인터페이스 클래스 : 추상함수 만

1.	추상클래스 와 인터페이스를 사용하는 이유 ? 
		자원을 상속해서 사용하려고 
		자원을 상속해서 사용하면 재사용이 아주 유용하다.
		자원을 상속해서 사용하면 자원 사용을 통일성 있게 유지할 수 있다. 
2.	추상클래스 와 인터페이스의 관계는 ?
		불완전한 추상클래스를 보완한 클래스가 인터페이스이다. 
		단일상속에서 다중상속 보완
		인터페이스에는 무조건 추상함수이다. 
3.	추상클래스는 실제 개발에서 많이 사용하지 않는다.
		java API에서 많이 사용한다. 
4. 실제 개발에서는 인터페이스를 사용한다. 
		초보때는 인터페이스 설계 금지 

5.	자바에서 상속을 하면 
	자식 클래스가 뚠뚠하다. 
	자식 클래스는 부모 클래스의 자원을 사용할 수 있다.

6.	자바를 즉시 인스턴스하는 new 키워드 보다 
	상속이 더 안정적이다. 
	현대 코드는 인터페이스 상속으로 발전하고 있다.
